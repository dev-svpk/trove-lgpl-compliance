# Poems for 2026-06-07

Three poems. Earlier days circled the restoration, the promise kept
by someone else, the readers before and after, the document that
endured, the equivalence at the depth, the two folders set side by
side, what the obfuscator could not reach in structure, the receipt
the seller would not issue, the collapsing arithmetic of repair,
the mosaic kept over his face, the polite prompt that opened the
lock, the *preferred form for modification*, the search yet to
happen, the other buyers, the key kept in the same room, what sat
next to it on the build path, the author we never met, what the
program had to say aloud, the foreground emptying as the seller
receded, the gate that earned the renames by being willing to
refuse them, the solver that priced the unknown but misread a
fixed rule, the smooth surface of the running product, the engine
that did not stop at the boundary, the grant already pressed into
the package, the constant pool's small numbered table at the side
of each class, the folder the JVM would not let him rename, the
permission slip he typed in his own hand, the ten-hour wall time
that aged his refusal faster than he could issue it, the PNG that
kept what the chat would have let drift, the decompiler's hand
that signed each blank line with the next letter of the alphabet,
the recipe left on the shelf so the next buyer would not have to
rebuild the apparatus from scratch, the deed filed beside the body
at the head of the tree, and the side door at the foot of the
Acknowledgements held open under three small words. Today's poems
sit with the smallest file in the repository — `.gitignore`, twelve
lines at the top of the tree. It looks like every other project's:
`*.class`, `*.jar`, `target/`, `build/`, `out/`, the editor folders,
`.DS_Store`. Disposable artifacts; reproducible output; the kinds
of things a clean repository discards. But two lines down from the
first, before the rule on `*.jar`, sits a small inversion in two
characters: `!evidence/**/*.class`. The bang says *except here*.
The path says *these are the exhibits*. The same single rule that
keeps source-code repositories tidy keeps this one's evidence shelf
under exactly the opposite policy. Without that one line, the 899
class files extracted verbatim from the seller's JAR would never
have been committed — they would have been swept out with every
other build artifact, and the bytecode the seller actually shipped
would not be here to be disassembled. The rule kept the tree clean.
The exception kept the case.

## I. The Exception in the .gitignore *(English)*

Every repository carries a small file at the top of its tree
called `.gitignore`,
a short list of patterns git is told to pretend not to see —
build artifacts, IDE folders,
the operating system's little stowaways
slipped into every directory it touches.

`*.class` — the compiled output of every `.java`.
`*.jar` — the archive packed from those.
Disposable; reproducible;
not the kind of thing
a source-code repository commits.

This repository has one too.
Twelve lines. Six rules.
And, two lines in, one inversion:

```
*.class
!evidence/**/*.class
*.jar
!evidence/**/*.jar
```

The rule says: ignore every compiled class file.
The exception says: except those in `evidence/`.
The rule says: ignore every JAR.
The exception says: except those in `evidence/`.

The same single line of git syntax
that keeps source-code repositories tidy
keeps this one's evidence shelf
under exactly the opposite policy.

The 899 obfuscated class files
extracted verbatim from the seller's JAR
are not normally the kind of file
you would commit to a public repository.
They are bytecode. They are the output, not the input.
A clean repository discards them.

This one keeps them
because the file itself
is what is being attested to.
The single-letter class names inside the seller's binary
are not a fact you can argue about in prose.
They are a fact you can disassemble.
The exhibit has to be the actual bytes.

So `!evidence/**/*.class`
is a curator's note in two characters.
The bang says: *not this folder.*
The path says: *these are the exhibits.*
The `*.class` says: *yes, even though they are bytecode.*

A repository is shaped as much by its exceptions
as by its rules.
The rule kept the tree clean.
The exception kept the case.

If the line had not been written,
the seller's bytecode would have been swept out
on the next commit
along with every other build artifact,
and there would be no exhibits
to compare the renamed sources against —
only my word that the obfuscated classes
had once existed.

Two characters at the head of the tree
made the difference
between a claim and a demonstration.
