by someone else, the readers before and after, the document that
endured, the equivalence at the depth, the two folders set side by
side, what the obfuscator could not reach in structure, the receipt
the seller would not issue, the collapsing arithmetic of repair,
the mosaic kept over his face, the polite prompt that opened the

next to it on the build path, the author we never met, what the

refuse them, the solver that priced the unknown but misread a

the one file the engine returned to its author still in single
letters where the courtesy reached its narrowest door. Today's
poems sit with what the library itself was built to be, and what
was specifically taken when its names were taken. Trove existed
because pre-generics Java had no way to say *set of int* or *map
from long to long* in a typed line. You could reach for
`Set<Integer>` and receive a container of boxed objects, with a
method signature that lied about what was inside; or you could

into the class name itself. `TIntHashMap`. `TLongHashSet`.
`TCharObjectMap`. Each class declared, in its own name, the
primitive it kept faith with. The vocabulary of the library *was*

container held. The obfuscator stripped all of it. The shapes
survived — the bytecode still pinned `iconst_0` to `int`,

ringing the same way. The restoration is not decoration. It is
putting back the one thing Trove had been built to carry: the
names that were the types, the types that were the names.

## I. Where the Names Were the Library *(English)*

Before Java had real generics,
you could write `Set<Integer>`
and receive a container of boxed objects
with a method signature that lied
about what was inside —

or you could reach for Trove
and receive `TIntHashSet`,
the primitive written
into the class name itself.

`TIntHashMap`. `TLongHashSet`. `TCharObjectMap`.
Each class declared, in its own name,
the primitive it kept faith with.
A reader knew, before the first method,
what the container held.

The vocabulary of the library
*was* its taxonomy.
The names were not labels for the work —
the names *were* the work.

The obfuscator stripped them.
`TIntHashMap` to `a`.
`TLongHashSet` to `b`.
`TCharObjectMap` to `c`.

The shapes survived.
The bytecode still pinned `iconst_0` to `int`,
`lconst_0` to `long`;
the method signatures still typed
their parameters honestly,
the JVM still knew which slot was four bytes
and which was eight.

But the labels —
the part of the library
that had been the point —
became thirty-some letters
all ringing the same way.

A library whose whole argument for existence
was the type written into the class name
became a wall of unmarked boxes
you had to open
to find out what was inside.

The taxonomy was still there
in the verbs.
The taxonomy was gone
from the nouns.

What the restoration puts back
is not decoration.
It is the one thing Trove had been built to carry:
the names that were the types,
the types that were the names,
the one-to-one
that was the library's only argument
for ever being there at all.

A reader opening `TIntHashMap.java`
now knows, before the first method,
what the container holds.
That sentence is the whole point of the library.
The seller had shipped it
with that sentence
deleted.
