# Poems for 2026-07-23

Three poems. Earlier days circled the restoration, the promise kept
by someone else, the readers before and after, the equivalence at
the depth, the two folders set side by side, the receipt the seller
would not issue, the mosaic kept over his face, the *preferred form
for modification*, the key kept in the same room, the constant
pool's small numbered table beside each class, the ten-hour wall
time that aged the seller's refusal faster than he could issue it,
the folder the JVM would not let him rename, the one file left in
single letters on the receiving side, the Coda's two small songs in
two languages, the month of silence between June and July, and the
return of the writing that ended it. Eight days have now passed
since that return without a further pause. Today's poems sit with
the image at the head of the README — the one PNG that shows the
running product, a poker solver caught in a still frame, felt
table, chips, community cards face up, an equities box in the
corner. That picture is what the seller sold. It is also the only
glimpse anywhere in this shelf of what the product looked like
while it ran. Everything else in the tree is what was in the frame
the whole time and did not appear in it: the maps and sets and
hash-tables of `gnu/trove/`, whose names the seller took off before
packing them underneath the picture. The screenshot is the front
of the record. The tree below it is what the screenshot did not
show.

---

## I. The Frame at the Head *(English)*

The README opens with a screenshot:
a poker solver on a bright screen,
felt-green table, chips, community cards face up,
an equities box in the corner.

One still frame from a running product.

That is what the seller sold.
That is what the buyer opened, on click.

Nothing in the frame shows the engine.
Under the surface, the tables of `gnu/trove/` —
the maps, the sets, the hash-tables,
each one wearing a single letter for a name —
did the work the equities box printed.
The frame did not need to mention them.
A solver that answers correctly
answers correctly
whether or not the engine is legible.

The screenshot is at the head of the README
because the screenshot is the object of the case:
not *here is my software*,
but *here is the software
in whose engine room the license lived,
and was gagged.*

Below the picture,
the file tree opens like a door.
`gnu/`, `evidence/`, `src-received/`, `LICENSE` —
each one a room the frame did not show.

A screenshot is small evidence.
It fits in a hundred kilobytes.
It says: *this ran; this was sold.*
The rest of the shelf answers:
*this is what was inside the frame,
with its names put back on.*
